Bug Report: Shadow matching fails with ClassCastException

Title: ClassCastException when matching join point shadows with generic types

Description:
The shadow matching mechanism throws a ClassCastException when trying to match certain types of join points.
This appears to be related to the handling of generic types in BcelShadow.

Error Message:
java.lang.ClassCastException: org.aspectj.weaver.ResolvedType cannot be cast to org.aspectj.weaver.ReferenceType
    at org.aspectj.weaver.patterns.SignaturePattern.matches(SignaturePattern.java:178)
    at org.aspectj.weaver.Shadow.match(Shadow.java:345)
    at org.aspectj.weaver.bcel.BcelShadow.matchesAt(BcelShadow.java:567)
    at org.aspectj.weaver.bcel.BcelClassWeaver.match(BcelClassWeaver.java:234)

Steps to Reproduce:
1. Define a pointcut with signature pattern including generic types
2. Apply aspect to class using those generic types
3. Execute weaving process

Expected Behavior:
Shadow matcher should correctly handle generic types and match join points

Actual Behavior:
ClassCastException is thrown during shadow matching

Related Classes:
- BcelShadow
- SignaturePattern
- ResolvedType
- ReferenceType
