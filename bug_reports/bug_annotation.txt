Bug Report: Annotation pattern matching fails

Title: WildAnnotationTypePattern throws RuntimeException with array values

Description:
When using annotation patterns that include array values, the pattern matcher
throws a RuntimeException indicating that array value support is not implemented.

Error:
java.lang.RuntimeException: Compiler limitation: annotation value support not implemented for type java.lang.String[]
    at org.aspectj.weaver.patterns.WildAnnotationTypePattern.resolveAnnotationValues(WildAnnotationTypePattern.java:231)
    at org.aspectj.weaver.patterns.WildAnnotationTypePattern.resolveBindings(WildAnnotationTypePattern.java:325)
    at org.aspectj.weaver.patterns.SignaturePattern.resolveBindings(SignaturePattern.java:81)

Example Code:
public aspect AnnotationAspect {
    pointcut annotated(): 
        execution(public * @MyAnnotation(values={"test1", "test2"}) com.example..*(..));
}

Where @MyAnnotation has:
@interface MyAnnotation {
    String[] values();
}

Expected Behavior:
Should be able to match on annotations with array values

Actual Behavior:
RuntimeException thrown, compilation fails

Affected Components:
- WildAnnotationTypePattern
- SignaturePattern
- Pattern matching system
